## Project Overview

**Avila Music** — e-commerce admin and storefront for a musical accessories store that sells primarily through MercadoLibre. Stack: Next.js (App Router) + Supabase (PostgreSQL + Auth + Storage + Edge Functions).

## Commands

```bash
# Install dependencies
npm install

# Dev server
npm run dev

# Generate Supabase TypeScript types (requires Supabase CLI and running local instance)
supabase gen types typescript --local > types/supabase.ts

# Apply migrations to local Supabase
supabase db reset   # wipes and re-applies all migrations + seed

# Run seed scripts manually (after migrations)
psql "$DATABASE_URL" -f supabase/seed/001_categories.sql
psql "$DATABASE_URL" -f supabase/seed/002_products.sql
psql "$DATABASE_URL" -f supabase/seed/003_config.sql

# Import historical sales from Excel
SUPABASE_URL=... SUPABASE_SERVICE_KEY=... python scripts/seed_sales.py
# Requires: pip install pandas openpyxl supabase
```

## Critical Business Rule — Frozen Price Snapshots

`sale_items.unit_price_usd` and `sale_items.unit_cost_usd` are **write-once at insert time**. They capture the product price and cost at the moment of sale. **Never update these fields** — doing so corrupts historical KPIs. `products.price_usd` and `products.cost_usd` hold the current catalog price and may change freely without affecting past sales.

## Database Architecture

### Category Hierarchy (2 levels, self-reference)

`categories` uses `parent_id` for a single self-join:
- `parent_id IS NULL` → root category (Guitarras, Bajo, Violín & Cuerdas, Batería & Percusión, Electrónica & Cables)
- `parent_id IS NOT NULL` → subcategory (leaf node)

**Products always reference a subcategory** (`products.subcategory_id` → leaf), never a root. This is a soft constraint enforced by convention — the schema does not technically block linking to a root, but all application code and seeds must link only to leaves.

### Computed Fields in `sale_items`

These fields are calculated at insert time in `scripts/seed_sales.py` (and must be in any future insert path):

```
commission   = price × ml_commission_rate
ship_cost    = shipping_cost_usd if free_shipping else 0
net_income   = price − commission − ship_cost
gross_profit = net_income − cost
margin_pct   = gross_profit / price
```

They are stored (not computed by the DB) so historical values remain stable if commission rates change.

### Key Views

| View | Purpose |
|---|---|
| `v_sale_items_detail` | Per-sale line items with full category hierarchy |
| `v_monthly_kpis` | Aggregated KPIs grouped by month (revenue, profit, margin) |
| `v_product_ranking` | Products ranked by units sold, with stock status |
| `v_category_tree` | Storefront navigation tree with `product_count` per subcategory |

### RLS Policy Pattern

All tables use `auth.role() = 'authenticated'` for full admin access. The public storefront gets read-only access via separate `public_read_*` policies scoped to `is_active = TRUE` on `categories` and `products`. All other tables (sales, sale_items, inventory, config) are admin-only.

## File Structure

```
supabase/
  migrations/001_initial_schema.sql  — all tables, triggers, views, RLS
  seed/
    001_categories.sql               — 5 roots then 13 subcategories (FK order matters)
    002_products.sql                 — 27 products mapped to subcategory slugs
    003_config.sql                   — app_config key-value pairs
scripts/
  seed_sales.py                      — imports historical sales from Excel sheets
lib/supabase.ts                      — typed Supabase client
types/supabase.ts                    — generated by Supabase CLI (do not edit manually)
```

## app_config Keys

| Key | Default | Meaning |
|---|---|---|
| `ml_commission_rate` | 0.11 | MercadoLibre commission (11%) |
| `shipping_cost_usd` | 1.30 | Cost absorbed per free-shipping sale |
| `lead_time_months` | 4 | Supplier lead time for reorder planning |
| `safety_stock_months` | 1 | Buffer stock target in months |
| `reinvestment_pct` | 0.70 | Share of net profit to reinvest |
| `personal_salary_pct` | 0.30 | Share of net profit as owner salary |

## Verification Queries

```sql
-- Should return 5 root categories, each with their subcategories and product_count
SELECT * FROM v_category_tree ORDER BY category_sort_order, subcategory_sort_order;

-- Should return rows for Feb, Mar, Apr 2026

```
